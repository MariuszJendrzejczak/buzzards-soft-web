Jak pracuję z AI

Kontrolowany pipeline,
nie „vibe-coding".

AI przyspiesza każdy etap wytwarzania — ale nic nie trafia dalej bez mojej decyzji. Ten sam tor za każdym razem, z wyraźnym podziałem: co robimy razem, co robi agent sam, a gdzie decyduje człowiek.

Pomysł · start
Kontekst i plan

Wspólnie ustalam pełny zakres zmiany: cel, obszar w kodzie, zależności i kolejność kroków. Kontekst projektu — konwencje, decyzje, architektura — jest już zapisany i podawany agentowi. Powstaje konkretny plan z jasno wyznaczonymi granicami tego, co wchodzi w zakres.

człowiek + AI
Akceptacja planu

Agent przedstawia gotowy plan, a ja przechodzę go punkt po punkcie — zatwierdzam kolejne kroki albo koryguję zakres. Jeśli na tym etapie coś się nie zgadza, wracamy do „Kontekst i plan" po nowe ustalenia. Kod rusza dopiero po akceptacji.

Human review

człowiek
Live testy manualne

Weryfikacja UI i UX na realnym urządzeniu — wychwytujemy to, czego agent nie oceni z samego kodu. Poprawki i polishing robimy wspólnie, aż całość działa i wygląda tak, jak ma trafić do wydania.

człowiek + AI
Wdrożenie

Praca wspólna oparta o CI/CD. Agent przygotowuje opis zmian i treść commita, wypycha gałąź feature i otwiera PR. W PR autonomiczny agent wpięty w GitHub robi niezależny review na podstawie diffa. Finalną akceptację, merge i uruchomienie deploymentu wykonuję ja.

Wdrożone
Warsztat pod maską

Trzy rzeczy, które sprawiają, że ten tor jest szybki i zdyscyplinowany.

Wyspecjalizowane sesje agentów

Osobny agent do planowania, budowy, testów i review — każdy z własnym zakresem i jednym deliverable. Nie jeden uniwersalny czat.

Realne narzędzia w stacku

Żywa dokumentacja bibliotek, weryfikacja UI w przeglądarce, testy e2e na mobile — agent ma dostęp do narzędzi, nie tylko do wiedzy.

Pamięć między sesjami

Kontekst projektu, decyzje i konwencje trwają dłużej niż jeden prompt — praca nie zaczyna się od zera każdego dnia.

Trzy zasady, na których stoi cały tor
Człowiek w pętli

Agent nie przekracza granicy plan → kod ani nie scala kodu bez mojej zgody.

Jedno źródło prawdy

Status pracy żyje w jednym miejscu; reszta widoku jest tylko jego lustrem.

Pętla, nie chaos

Regresje domykają ten sam tor, zamiast rozsypywać pracę na równoległe wątki.
